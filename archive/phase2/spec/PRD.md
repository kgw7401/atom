# ATOM — Product Requirements Document

> 훈련을 돕고, 성장을 보이게 만든다.

Version 4.0 | February 2026

---

## 1. Why Atom Exists

### 1.1 문제

복싱을 배우는 사람은 누구나 같은 벽을 만납니다.

처음 3개월은 모든 게 새롭습니다. 잽, 스텝, 가드. 매주 뭔가 배운다는 느낌이 강합니다. 성장이 자명합니다.

6개월이 지나면 달라집니다. 기본기는 익혔는데, 코치가 "좋아졌어"라고 해도 뭐가 좋아진 건지 모르겠습니다. 스파링을 하면 여전히 같은 실수를 반복합니다. 매일 같은 루틴을 돌리는데, 이게 맞는 방향인지 확신이 없습니다.

1년이 지나면 두 갈래로 갈립니다. 계속하는 사람과 그만두는 사람. 그만두는 사람의 이유는 대부분 "재미없어서"인데, 그 재미없음의 실체는 **"내가 제자리인 것 같다"**는 느낌입니다.

### 1.2 문제의 본질

복싱 훈련생이 그만두는 근본 원인은 기술 부족이 아닙니다. **성장 인지 실패**입니다.

- **뭘 해야 하는지 모릅니다.** 코치가 없는 시간에는 늘 하던 걸 반복합니다. 익숙한 걸 반복하니 안 늡니다.
- **하고 있는 게 맞는지 모릅니다.** 어퍼컷을 300번 쳤는데, 그 300번이 제대로 된 건지 확인할 방법이 없습니다.
- **늘고 있는지 모릅니다.** "어제의 나 vs 오늘의 나"를 비교할 도구가 없습니다. 코치의 "좋아졌네" 외에는 근거가 없습니다.

이 세 가지가 결합하면 "열심히 하는데 제자리"라는 최악의 감정이 만들어집니다.

> 복싱의 성장 곡선은 로그함수처럼 생겼는데, 사람의 기대는 선형입니다. 실제로 늘고 있어도 느리게 느껴져서 좌절합니다.

### 1.3 Atom의 답

Atom은 복싱 기술을 가르치는 앱이 아닙니다. **나를 아는 코치**입니다.

좋은 코치는 네 가지를 해줍니다:

- **오늘 뭘 해야 하는지 알려줍니다.** "너는 잽-크로스만 치잖아. 오늘은 바디 콤보 섞어보자."
- **옆에서 같이 훈련합니다.** "원-투! 원-투-바디! 좋아, 계속!"
- **뭐가 달라졌는지 짚어줍니다.** "2주 전에는 원-투만 쳤는데, 이제 6가지 콤보를 쓰잖아."
- **다음에 뭘 할지 잡아줍니다.** "콤보는 됐으니까, 이제 쉐도우에서 실전처럼 써보자."

Atom은 이 코치의 역할을 확장합니다. 코치가 없을 때도, 혼자 훈련할 때도, 이 역할이 계속되게 합니다.

> 코치를 대체하는 게 아닙니다. 코치가 못 해주는 것을 해줍니다: 매 훈련의 기록, 장기간 변화 추적, 객관적 비교, 혼자 훈련할 때의 구조화된 가이드.

## 2. 핵심 루프

Atom의 모든 것은 하나의 루프입니다:

```
세션 생성 (내 데이터 기반)
    ↓
실제 훈련 수행 (TTS 코치 가이드)
    ↓
영상 (훈련의 증거 + 데이터 소스)
    ↓
분석 → 피드백 (코치 언어로)
    ↓
데이터 누적 → 다음 세션 생성
    ↓
처음으로
```

**내 데이터 기반 훈련을 하고, 그 훈련이 다시 데이터가 되어 다음 훈련을 만듭니다.**

이 루프가 반복될수록 세션은 더 정확해지고, 피드백은 더 구체적이 되고, 사용자는 실제로 성장합니다.

## 3. Who Is Atom For

| 항목 | 정의 |
|------|------|
| 훈련 경력 | 복싱 3개월 ~ 3년. 기본기는 익혔지만 성장이 정체된 느낌을 받는 사람 |
| 훈련 빈도 | 주 2회 이상. 헬스장 또는 자택에서 훈련 |
| 동기 | 실력 향상에 진지한 관심. "나아지고 있다"는 객관적 확인을 원함 |
| 환경 | 코치가 있을 수도 없을 수도 있음. 혼자 훈련하는 시간이 존재 |
| 핵심 감정 | "열심히 하는데 제자리인 것 같다" |

**Atom의 포지션: 코치를 대체하는 게 아니라, 코치가 못 해주는 것을 해주는 도구**

| 코치가 잘 하는 것 | Atom이 잘 하는 것 |
|-------------------|-------------------|
| 실시간 폼 교정 | 매 훈련의 데이터 기록 및 추적 |
| 스파링 파트너 | 장기간 변화의 객관적 비교 |
| 동기 부여, 멘탈 코칭 | 혼자 훈련할 때의 구조화된 가이드 |
| 경험 기반 직관 | 데이터 기반 성장 증명 |

## 4. 두 개의 세션

Atom에는 두 가지 훈련 세션이 있습니다. 각각의 목적이 다릅니다.

### 4.1 콤비네이션 드릴 — "무기를 만든다"

**목적:** 새로운 콤보를 배우고 몸에 익히기.

복싱에서 초급→중급의 벽은 레퍼토리입니다. 잽-크로스만 치던 사람이 훅을 섞고, 바디를 섞고, 3타·4타 콤보를 쓰기 시작하는 것. 이건 폼의 문제가 아니라 **경험의 문제**입니다. 안 해본 건 쓸 수 없습니다.

**구조:** 지시형. TTS 코치가 콤보를 직접 호출합니다.

```
"원-투!"
"원-투-쓰리!"
"원-투-바디!"
"원-투-쓰리-투!"
"잽-바디-잽-크로스!"
```

익숙한 콤보 사이에 새 콤보를 자연스럽게 끼워넣는 방식입니다. 갑자기 어려운 걸 주는 게 아니라, 할 수 있는 것 → 약간 새로운 것 → 할 수 있는 것 → 약간 새로운 것. 이 리듬으로 구성합니다.

**측정:**

- 어떤 펀치를 쳤는지 (잽/크로스/훅/어퍼컷/바디)
- 지시한 콤보를 실제로 수행했는지
- 콤보별 성공률
- 콤보별 빈도 분포

**피드백 예시:**

- "원-투-쓰리를 10번 지시했는데 7번 성공했어요."
- "바디훅은 아직 어색한 것 같아요. 10번 중 3번만 바디로 갔어요."
- "원-투는 완전히 자연스러워요. 20번 중 20번."

**다음 세션 생성 로직:**

- 성공률 높은 콤보 → 비중 줄이기
- 성공률 낮은 콤보 → 비중 늘리기
- 충분히 익숙해지면 → 새 콤보 하나 추가

**성장 지표:** "쓸 수 있는 콤보 수"

- 2주 전: 원-투만 성공률 80% 이상
- 오늘: 6가지 콤보가 성공률 80% 이상

이건 설명이 필요 없는 성장입니다.

### 4.2 쉐도우 복싱 — "무기를 실전에서 꺼내 쓴다"

**목적:** 배운 콤보를 실전 상황에서 선택하고 적용하기.

쉐도우 복싱의 문제는 대부분 **"혼자 허공에 주먹을 휘두르는 것"**이 된다는 점입니다. 상대가 없으니까 그냥 아는 콤보만 반복합니다. 코치가 "동작이 제한적이다"라고 하는 이유입니다.

좋은 쉐도우 복싱은 "머릿속에 상대를 두고 하는 것"입니다. 하지만 초중급자는 뭘 상상해야 하는지 모릅니다. **Atom이 상상 속의 상대를 만들어줍니다.**

**구조:** 반자유형. TTS 코치가 상황을 호출하고, 사용자가 반응을 선택합니다.

```
"상대가 잽을 쐈어!"        → 슬립 후 카운터? 백스텝 후 잽? 파링 후 바디?
"상대가 거리를 좁혀!"      → 클린치? 백스텝? 어퍼컷?
"상대가 가드를 내렸어, 찬스!" → 자유 콤보 선택
"상대가 바디를 노려!"      → 엘보우 블록 후 반격?
"상대가 물러나!"          → 추격 콤보? 거리 유지?
```

콤비네이션 드릴과의 핵심 차이: 드릴은 **"이걸 쳐"**이고, 쉐도우는 **"상황이야, 알아서 반응해"**입니다.

**측정:**

| 지표 | 설명 | 왜 중요한가 |
|------|------|------------|
| 공방 비율 | 상대 공격 시나리오에서 방어 반응을 했는가 | 공격만 하면 실전에서 맞기만 함 |
| 반응 다양성 | 같은 상황에 다른 반응을 하는가 | 반응이 예측 가능하면 스파링에서 읽힘 |
| 콤보 선택 적절성 | 상황에 맞는 콤보를 선택하는가 | 추격 찬스에 잽 하나만 치면 찬스를 날림 |
| 콤보 다양성 | 자유 상황에서 얼마나 다양한 콤보를 쓰는가 | 드릴에서 배운 것을 실전에서 꺼내 쓰는가 |

**피드백 예시:**

- "상대 공격 시나리오가 15번 나왔는데, 방어 반응은 9번이었어요. 6번은 무시하고 공격만 계속했어요. 실전에서는 맞는 상황이에요."
- "상대 잽에 대한 반응이 매번 백스텝이었어요. 슬립이나 파링도 섞어보세요. 반응이 예측 가능하면 스파링에서 읽혀요."
- "추격 찬스에서 잽 하나만 치고 끝낸 게 4번 있었어요. 드릴에서 배운 원-투-쓰리를 써보세요."
- "드릴에서는 8가지 콤보를 수행할 수 있는데, 쉐도우에서는 3가지만 쓰고 있어요."

**성장 지표:** "실전에서 쓰는 콤보 수" + "상황 대응력"

### 4.3 두 세션의 관계

```
Phase 1: 콤비네이션 드릴
"쓸 수 있는 콤보: 2개 → 8개"
    ↓
Phase 2: 쉐도우 복싱
"자유 상황에서 실제로 쓰는 콤보: 2개 → 6개"
```

드릴에서 무기를 만들고, 쉐도우에서 무기를 실전처럼 꺼내 씁니다.

드릴에서 8개를 배워도 쉐도우에서 다 쓰진 못합니다. 익숙한 것만 나옵니다. 이 갭을 줄이는 게 쉐도우 세션의 역할입니다. 그리고 이 **드릴 성공률 vs 쉐도우 사용률의 갭** 자체가 강력한 인사이트입니다:

> "드릴에서 바디 콤보 성공률 80%인데, 쉐도우 자유 상황에서 바디 콤보 사용률 5%. 알면서도 안 쓰고 있어요."

이걸 의식하기 시작하면 진짜 늘기 시작합니다.

## 5. Product Experience

### 5.1 Onboarding: 5분 안에 첫 훈련까지

> 원칙: 첫 훈련을 경험하는 것이 와우포인트입니다.

**Step 1 — 자가진단 (2분)**

4개 질문으로 현재 상태와 목표를 파악합니다.

- 훈련 경력: 3개월 미만 / 3개월~1년 / 1년+
- 주로 하는 훈련: 쉐도우 / 헤비백 / 스파링 / 혼합
- 가장 부족하다고 느끼는 영역: 공격 다양성 / 방어 / 체력 / 리듬감 / 기술 정확도 (복수 선택)
- 가장 고치고 싶은 것 하나

**Step 2 — 영상 업로드 (선택)**

"기존 훈련 영상이 있으면 올려보세요. 더 정확한 첫 세션을 만들어드려요."

- 영상 있으면: AI가 현재 콤보 레퍼토리 파악 → 더 정밀한 첫 세션
- 영상 없으면: 자가진단 기반으로 첫 세션 생성

**Step 3 — 첫 콤비네이션 드릴 (즉시)**

바로 시작합니다. 자가진단 결과를 기반으로 맞춤화된 드릴 세션.

이 첫 세션이 와우포인트: **"아, 이게 혼자 하는 거랑 다르구나."**

**Step 4 — 첫 피드백**

- 촬영했다면: "원-투는 자연스러운데, 훅은 아직 어색해요. 10번 중 4번 성공."
- 촬영 안 했다면: 세션 완료 기록 + "다음에 촬영하면 콤보별 성공률을 알려드릴 수 있어요."
- 즉시 다음 세션 제안

### 5.2 Daily Loop

온보딩 이후 모든 훈련은 이 루프를 따릅니다.

**① 세션 확인 — "오늘 뭘 할지"**

앱을 열면 오늘의 세션이 준비되어 있습니다.

> "오늘은 바디 콤보 드릴이에요. 저번에 바디훅 성공률이 30%였어요. 오늘 50%까지 올려봅시다."

**② 훈련 수행**

세션 시작 버튼을 누르면 TTS 코치가 시작됩니다.

- 📹 **촬영 포함 (기본, 권장)**: TTS 코칭 + 자동 촬영. 완전한 피드백 가능.
- 🔊 **오디오 온리 (대안)**: 촬영 환경이 안 될 때. TTS 코칭만. 피드백 제한적.

세션 중 촬영의 핵심 가치: **"지시한 것 vs 실제 수행"의 갭**을 알 수 있습니다.

**③ 피드백**

세션이 끝나면 코치 언어로 피드백을 줍니다. (피드백 상세 규칙은 섹션 6 참조)

**④ 다음 세션 자동 생성**

피드백 데이터를 기반으로 다음 세션이 자동으로 준비됩니다. 사용자는 "다음에 뭘 하지?"를 고민할 필요가 없습니다.

### 5.3 성장 가시화

Atom은 두 단계로 성장을 보여줍니다.

**1단계: 콤보 레퍼토리 (드릴 기반)**

"쓸 수 있는 콤보"가 늘어나는 것을 추적합니다.

```
나의 콤보 레퍼토리:

✅ 원-투           성공률 95%  — 완전히 익숙함
✅ 원-투-쓰리       성공률 85%  — 익숙함
✅ 원-투-바디       성공률 72%  — 거의 다 됨
🔄 잽-바디-잽-크로스  성공률 45%  — 연습 중
🔄 원-투-쓰리-투    성공률 30%  — 아직 어색함
🆕 원-투-슬립-크로스  아직 안 해봄
```

> "2주 전: 성공률 80% 이상 콤보 2개 → 오늘: 5개. 쓸 수 있는 무기가 늘고 있어요."

**2단계: 실전 적용 (쉐도우 기반)**

드릴에서 배운 콤보를 자유 상황에서 쓰는지 추적합니다.

```
드릴 vs 쉐도우 비교:

원-투          드릴 95% → 쉐도우 사용률 40%  — 자주 씀
원-투-쓰리     드릴 85% → 쉐도우 사용률 25%  — 가끔 씀
원-투-바디     드릴 72% → 쉐도우 사용률 5%   — 거의 안 씀 ⚠️
```

> "드릴에서 바디 콤보 성공률 72%인데, 쉐도우에서는 5%만 사용. 몸은 할 수 있는데 실전에서 안 꺼내고 있어요."

### 5.4 Growth Project: 한 가지에 집중

성장 데이터에서 가장 시급한 한 가지를 골라 집중하는 기간입니다.

- 예: "바디 콤보 레퍼토리 확장" — 2~3주간 이 영역에 집중
- 매 세션의 드릴이 이 목표에 맞춤화됨
- 세션마다 해당 콤보의 변화를 추적

**프로젝트 완료 시 — Before/After:**

> "3주 전: 바디 콤보 성공률 0%. 바디샷을 한 번도 안 쳤어요."
> "오늘: 바디 콤보 3가지 성공률 70% 이상. 쉐도우에서도 바디를 섞기 시작했어요."

이 순간이 **"진짜 늘었다"를 느끼는 피크 모먼트**입니다.

완료 후 다음 프로젝트가 자동 제안됩니다.

### 5.5 Home: 달력 + 현재 상태 + 다음 세션

Home은 세 가지를 보여줍니다.

**① 달력 — 훈련 기록**

이번 달 세션 기록이 달력에 표시됩니다:

- 2/25: 콤비네이션 드릴 12분 — "바디훅 성공률 30% → 45%"
- 2/27: 쉐도우 복싱 15분 — "상대 공격 시 방어 반응 60%"
- 2/28: (오늘) 세션 예정

세션이 기록되고, 각 세션의 한줄 요약이 달리면서, 훈련 빈도와 흐름이 보입니다.

**② 현재 상태**

- 🎯 현재 집중: "바디 콤보 레퍼토리 확장"
- ✅ 쓸 수 있는 콤보: 5개 (2주 전: 2개)
- 🔄 연습 중: 잽-바디-잽-크로스 (성공률 45%)

**③ 다음 세션 — CTA**

"오늘의 훈련: 바디 콤보 드릴 12분" — 바로 시작할 수 있습니다.

**Cold Start (첫 방문):**

달력은 비어있고, 데이터도 없습니다. 단일 CTA: "첫 훈련 시작하기" → 온보딩.

## 6. 피드백 언어 규칙

### 원칙

- **구체적**: "공격이 단조로워요"가 아니라 "원-투만 15번 쳤어요. 바디 콤보를 한 번도 안 썼어요."
- **내 얘기**: 내 영상에서 나온, 나의 이전 기록과 비교한 피드백
- **행동 가능**: 다음 훈련에서 바로 의식할 수 있는 수준
- **변화 중심**: 이전과 비교해서 뭐가 달라졌는지
- **절제된 격려**: 과하지 않은 긍정

### 콤비네이션 드릴 피드백 예시

| 상황 | 피드백 |
|------|--------|
| 첫 세션 후 | "원-투는 자연스럽게 나와요. 훅은 10번 중 4번 성공. 바디샷은 아직 안 해봤어요." |
| 성공률 향상 | "원-투-쓰리 성공률이 30%에서 70%로 올랐어요. 확실히 익숙해지고 있어요." |
| 새 콤보 도전 | "오늘 처음 해본 원-투-슬립-크로스, 10번 중 2번 성공. 첫날치고 괜찮아요. 반복이 필요해요." |
| 지시 vs 수행 갭 | "바디훅을 지시했는데, 실제로는 헤드훅으로 간 게 7번 있었어요. 바디를 의식적으로 노려보세요." |

### 쉐도우 복싱 피드백 예시

| 상황 | 피드백 |
|------|--------|
| 방어 미반응 | "상대 공격 시나리오 15번 중 6번을 무시하고 공격만 계속했어요. 실전에서는 맞는 상황이에요." |
| 반응 단조 | "상대 잽에 매번 백스텝이었어요. 슬립이나 파링도 섞어보세요. 예측 가능하면 읽혀요." |
| 찬스 미활용 | "추격 찬스에서 잽 하나만 치고 끝낸 게 4번. 드릴에서 배운 원-투-쓰리를 꺼내보세요." |
| 드릴 vs 쉐도우 갭 | "드릴에서는 8가지 콤보를 쓸 수 있는데, 쉐도우에서는 3가지만 나왔어요." |

### 금지 표현

- 내부 수치 노출: "Offensive Profile: 0.34"
- 막연한 진단: "방어가 약합니다"
- 기술 용어: "repertoire_entropy", "combo_diversity_index"
- 과한 긍정: "대단해요!", "최고예요!"

## 8. Implementation Roadmap

### Phase 1: 콤비네이션 드릴 루프 (4주)

첫 드릴 세션을 경험하고, 콤보 레퍼토리가 쌓이는 것까지.

| # | 항목 | 예상 |
|---|------|------|
| 1 | 온보딩 (자가진단 + 영상 선택) | 3일 |
| 2 | 첫 드릴 세션 생성 (자가진단 기반) | 2일 |
| 3 | Drill Session 화면 (TTS + 타이머 + 촬영/오디오 온리) | 5일 |
| 4 | 영상 분석: 펀치 분류 + 콤보 인식 | 5일 |
| 5 | 피드백 화면 (콤보별 성공률, 지시 vs 수행) | 3일 |
| 6 | 다음 세션 자동 생성 (데이터 기반) | 2일 |
| 7 | Home: 달력 + 콤보 레퍼토리 + 다음 세션 CTA | 3일 |
| 8 | Growth Project (콤보 집중) 데이터 모델 + UI | 3일 |

### Phase 2: 쉐도우 복싱 세션 (3주)

배운 콤보를 실전처럼 꺼내 쓰는 훈련까지.

| # | 항목 | 예상 |
|---|------|------|
| 9 | Shadow Session 화면 (상황 호출 TTS + 촬영) | 5일 |
| 10 | 쉐도우 분석: 공방 비율, 반응 다양성, 콤보 선택 | 4일 |
| 11 | 쉐도우 피드백 화면 | 3일 |
| 12 | 드릴 vs 쉐도우 갭 분석 + 피드백 | 2일 |
| 13 | Combo Repertoire 화면 (드릴 성공률 vs 쉐도우 사용률) | 3일 |

### Phase 3: Growth Proof (2주)

성장을 체감하고, 다음 목표로 이어지는 것까지.

| # | 항목 | 예상 |
|---|------|------|
| 14 | 프로젝트 완료 조건 판단 | 2일 |
| 15 | Before/After 스토리 화면 | 3일 |
| 16 | 다음 프로젝트 자동 제안 | 2일 |
| 17 | 달력 뷰 히스토리 + 장기 추이 | 3일 |

**총 예상 기간: 약 9주**

## 10. Risks

| 리스크 | 영향 | 완화 |
|--------|------|------|
| 펀치 분류 정확도 부족 | 콤보 성공률이 엉뚱하게 나옴 → 신뢰 상실 | 높은 confidence만 피드백. 불확실한 건 제외 |
| TTS 부자연스러움 | "코치와 함께" 느낌 감소 | MVP: 사전 녹음 오디오. 향후: Custom Voice |
| 촬영 환경 제약 | 데이터 부족 → 루프 약화 | 오디오 온리 모드 + 촬영 시 더 나은 경험으로 유도 |
| 분석 지연 | 세션 후 피드백 대기 → 이탈 | 즉시 세션 요약 + 상세 분석은 백그라운드 |
| 쉐도우 상황 인식 한계 | 상대 공격에 방어했는지 판단 어려움 | 타이밍 기반 휴리스틱 + 점진적 모델 개선 |
| 콤보 난이도 커브 | 너무 쉬우면 지루, 너무 어려우면 좌절 | 성공률 기반 자동 난이도 조절 |

## 11. Design Invariants

Atom에서 반드시 지켜야 하는 원칙:

**1. Training-first.**
앱의 모든 경로는 "훈련 시작"으로 수렴합니다. 분석이나 리포트는 훈련의 부산물입니다.

**2. 코치의 말로.**
모든 피드백은 코치가 하는 말처럼 전달됩니다. 내부 수치나 기술 용어가 사용자에게 노출되지 않습니다.

**3. 내 얘기여야 한다.**
일반적인 복싱 조언이 아니라, 내 데이터에서 나온 피드백이어야 합니다.

**4. 연속된 루프.**
훈련 → 피드백 → 다음 훈련이 끊김 없이 이어집니다.

**5. 마찰 최소화.**
사용자가 하는 건 "세션 시작" 하나뿐입니다.

**6. 무기 먼저, 실전은 그 다음.**
콤보를 몸에 익히고(드릴), 그 다음에 실전처럼 꺼내 쓰는(쉐도우) 순서를 지킵니다.

**7. 5분 안에 첫 훈련.**
앱 설치부터 첫 세션 시작까지 5분을 넘지 않습니다.
