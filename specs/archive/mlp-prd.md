## 2.1 Problem Deep Dive

**Who is affected?**

```
┌─ Persona A ────────────────────────────────────────────────────┐
│ Name: 복싱 입문자 (1-6개월)                                    │
│ Context: 체육관 다니기 시작했지만 혼자 연습할 때 뭘 해야 할지   │
│          모름. 유튜브 보면서 하지만 체계가 없음.                │
│ Pain level: 🔴 high                                           │
│ Current workaround: 유튜브 영상 따라하기, 기억나는 것만 반복   │
└────────────────────────────────────────────────────────────────┘

┌─ Persona B ────────────────────────────────────────────────────┐
│ Name: 취미 복서 (6개월+)                                       │
│ Context: 기본기는 있지만 성장 정체기. 매일 같은 루틴 반복.      │
│          실력이 늘고 있는지 체감이 안 됨.                       │
│ Pain level: 🟡 medium                                         │
│ Current workaround: 체육관 코치에게 가끔 물어봄, 자체 기록 없음│
└────────────────────────────────────────────────────────────────┘
```

**What happens if we do nothing?**
- 사용자가 1-2주 사용 후 "매번 같은 느낌"으로 이탈
- 성장 실감이 없으면 유료 전환 동기 없음
- 복싱 앱 시장에서 단순 타이머 앱과 차별화 실패


## 2.2 Solution Details

- **Direction:** Deep Coach — AI 코치를 핵심 차별화 요소로
- **Core concept:** 매 세션에서 코치가 사용자의 히스토리를 기반으로 맞춤 훈련을 설계하고, 세션 전후로 개인화된 피드백을 제공한다. 시간이 지나면 사용자는 자신의 성장 궤적을 시각적으로 확인할 수 있다.

[스트릭/마일스톤]       [대화형 피드백]       [목표 달성 시각화]   [코치가 교정]

### Phase 1: MLP v1 (완료)

이미 구현 완료. 핵심 변경:
- LLM 프롬프트에 코치 캐릭터 성격 부여 (반말, 격려, 히스토리 참조)
- 세션 전 코치 메시지, 세션 후 리뷰 (완료/중단 분기)
- 스트릭 계산 + 마일스톤 자동 감지 + 축하 UI
- 프로필: 12주 히트맵, 콤보 숙련도, 마일스톤 배지

### Phase 2: Coach Depth — "내 코치" 만들기

코치를 "기능"에서 "캐릭터"로 진화. **핵심 원칙: 세션 단위 경험 강화.**
Phase 2는 개별 세션의 감정적 깊이를 높이는 데 집중한다. 장기 데이터 분석(주간/월간)은 Phase 3.

| Feature | Description | Phase 2인 이유 |
|---------|-------------|----------------|
| 코치 이름/성격 | 사용자가 코치 스타일 선택 (엄격/친근/분석적). 프롬프트에 반영 | 세션마다 일관된 캐릭터 |
| 온보딩 플로우 | 첫 실행 시 경험 레벨, 목표, 선호 스타일 수집. 코치 첫 인사 | 첫 세션 경험 |
| 햅틱/사운드 피드백 | 세션 완료 시 진동, 마일스톤 달성 시 사운드 | 세션 중/후 감각 피드백 |
| 세션 전후 코칭 강화 | "어제 잽-크로스 많이 했으니 오늘은 훅 위주로 가자" | 직전 세션 기반 단기 조언 |
| 이벤트 트래킹 기반 | Phase 3 지표 측정을 위한 analytics 인프라 구축 | 측정 없이 개선 불가 |

### Phase 3: Growth System — "진짜 성장했다"

사용자가 장기적 성장을 체감하는 시스템. **핵심 원칙: 다중 세션 데이터 분석.**
Phase 3는 축적된 데이터를 바탕으로 성장 서사를 만든다. Phase 2의 이벤트 트래킹이 선행 조건.

| Feature | Description | Phase 3인 이유 |
|---------|-------------|----------------|
| 주간 리포트 | LLM이 주간 훈련 데이터 요약 + 코치 코멘트 생성 | 7일치 데이터 필요 |
| 훈련 프로그램 | 4주/8주 구조화 프로그램. 코치가 진도에 맞게 조절 | 장기 목표 기반 플래닝 |
| 목표 시각화 | 목표 대비 진척도 게이지, 예상 달성일 | 누적 데이터 시각화 |
| 콤보 추천 진화 | 숙련도 기반 자동 난이도 조절 | combo_exposure 축적 필요 |

### Phase 4: Form Feedback (Track B 연동)

기술적으로 가장 도전적. Track B 연구 결과를 제품에 통합:

| Feature | Description |
|---------|-------------|
| 카메라 폼 분석 | 세션 중 카메라로 자세 캡처 → 포즈 분석 |
| 코치 폼 피드백 | VLM이 세션 후 폼 리뷰 생성, 코치 캐릭터로 전달 |
| Track B 의존 | B1 Pose Pipeline → B2 Action Recognition 완성 필요 |

**What this is NOT** (explicit out-of-scope):
- 실시간 대전/매칭 기능
- 체육관 예약/결제 시스템
- 식단/체중 관리
- 다른 격투기 종목 (킥복싱, MMA 등)

**측정 인프라 요구사항:**

Phase 1에서 이미 측정 가능한 지표(#1, #2, #3, #5)는 서버 DB 쿼리로 확인.
Phase 2에서 추가해야 할 것:
- **클라이언트 이벤트 트래킹**: 화면 체류 시간, 버튼 탭, 세션 플로우 전환
- **인앱 설문**: 세션 10회 달성 후 NPS 또는 간단 만족도 팝업
- 초기에는 자체 이벤트 테이블로 시작, 이후 Firebase Analytics 등 전환 가능

## 2.5 Cost Model

```
┌─────────────┬──────────────────────┬─────────────┬────────────────────────┐
│ Phase       │ LLM 호출             │ 세션당 비용  │ 월 비용 (주3회 유저)   │
├─────────────┼──────────────────────┼─────────────┼────────────────────────┤
│ Phase 1-2   │ 1회/세션 (플랜 생성) │ ~$0.002     │ ~$0.024                │
│ Phase 3     │ +1회/주 (주간 리포트)│ ~$0.004     │ ~$0.052                │
│ Phase 4     │ +1회/세션 (폼 리뷰)  │ ~$0.010     │ ~$0.130                │
└─────────────┴──────────────────────┴─────────────┴────────────────────────┘
```

**유료화 연결 포인트:**
- Phase 1-2: 무료로 제공 가능한 비용 수준 (유저당 월 $0.03 미만)
- Phase 3: 주간 리포트를 프리미엄 기능으로 분리 가능
- Phase 4: 폼 분석은 명확한 유료 가치 제안 (비용도 가장 높음)
- **원칙**: 코치 기본 경험은 무료, 성장 분석 도구는 유료
